# Evaluation Results Comparison

This script combines results from three different evaluation systems to create comparison files for analysis.

## Overview

The comparison script processes outputs from:
1. **OpenAI Direct** (`gpt-5-mini`) - Direct OpenAI API calls with only event list data
2. **PLLM Meta-Analyst Only** - AstroMind system with only the metadata analyst enabled
3. **PLLM Full Multi-Agent** - AstroMind system with all agents enabled

## Input Files

The script requires the following input files:

1. **OpenAI Results**: `outputs/openai/openai_results_0_100.json`
   - Contains direct OpenAI GPT-5-mini responses
   - Generated by `bulk_openai_direct.py`

2. **PLLM Meta-Only Results**: `outputs/only_meta/combined_results.json`
   - Contains results from metadata analyst only configuration
   - Generated by `bulk_metadata_analysis.py` with only `metadataAnalyst` enabled

3. **PLLM Full Results**: `outputs/all_models/combined_results.json`
   - Contains results from full multi-agent system
   - Generated by `bulk_metadata_analysis.py` with all agents enabled

4. **Input Sources**: `input_sources.json`
   - Contains source metadata from Chandra Source Catalog
   - Generated by `get_data/extract_from_mongodb.py`

## Usage

### Quick Start

```bash
./run_comparison.sh
```

### Manual Execution

```bash
cd scripts/bulk_eval
python3 create_comparison.py
```

## Features

- **Automatic Source Matching**: Matches sources across all files by their `_id` or `source_id`
- **Missing Data Handling**: Gracefully handles sources that are missing from one or more result files
- **Coverage Statistics**: Reports coverage for each evaluation system
- **Catalog Data Extraction**: Extracts relevant catalog metadata while excluding raw data fields

## Excluded Fields

The following fields are excluded from the `chandra_source_catalog_data` section to avoid redundancy:

- `_id` (already in top level)
- `source_name` (already in top level)
- `obsid` (already in top level)
- `event_list` (raw data, not metadata)
- `original_event_list` (raw data, not metadata)
- `pca_64d` (processed feature, not catalog data)
- `umap_2d` (processed feature, not catalog data)
- `neighbors` (separate enrichment, not catalog data)

## Output Statistics

The script provides detailed statistics:
- Total unique sources across all files
- Number of sources with complete comparison data
- Number of sources skipped due to missing data
- Coverage percentage for each evaluation system

## Error Handling

The script handles various error conditions:
- Missing input files (warns and continues)
- Invalid JSON format (reports error and exits)
- Missing source metadata (warns and skips source)
- Sources without any answers (warns and skips)
